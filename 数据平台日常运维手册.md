## 三、日常启动与停止

### 3.1 完整启动（推荐顺序）

```bash
# ===== 第一步：启动 Airbyte（kind k8s 集群） =====
# Airbyte 容器在 Docker 重启后通常会自动恢复
# 如果没有自动恢复，执行：
docker start airbyte-abctl-control-plane

# 验证 Airbyte 是否正常（需等待 2-3 分钟）
docker exec airbyte-abctl-control-plane kubectl get pods -A
# 所有 pod 状态应为 Running

# ===== 第二步：启动数据平台所有服务 =====
cd /home/zp/airbyte/docker-platform
docker compose up -d

# ===== 第三步：验证所有服务 =====
docker compose ps
# 所有容器应显示 Up 状态
```

### 3.2 快速启动（服务器重启后）

```bash
# 一条命令启动所有业务容器
docker start source-postgres dest-postgres minio mlflow metabase ollama

# 如果使用 docker-compose 管理：
cd /home/zp/airbyte/docker-platform && docker compose up -d
```

### 3.3 停止所有服务

```bash
# 方式1：docker compose 停止（推荐，不丢数据）
cd /home/zp/airbyte/docker-platform
docker compose stop

# 方式2：使用管理脚本
cd /home/zp/airbyte/docker-platform
./run-pipeline.sh stop

# 方式3：手动停止单个服务
docker stop metabase mlflow minio ollama dest-postgres source-postgres
```

### 3.4 停止 Airbyte

```bash
# 仅停止 Airbyte（不影响其他服务）
docker stop airbyte-abctl-control-plane

# 注意：停止 Airbyte 不影响已同步的数据，只是暂停同步功能
```

### 3.5 完全关机（长期不用时）

```bash
# 1. 停止业务服务
cd /home/zp/airbyte/docker-platform && docker compose stop

# 2. 停止 Airbyte
docker stop airbyte-abctl-control-plane

# 数据全部在 Docker Volume 中持久化，重新启动后恢复
```

## 四、各模块运维操作详解

### 4.1 Airbyte — 数据同步管理

```bash
# --- 启动 ---
docker start airbyte-abctl-control-plane
# 等待 2-3 分钟后访问 http://localhost:8000

# --- 检查状态 ---
docker exec airbyte-abctl-control-plane kubectl get pods -A
# 期望看到所有 pod 为 Running 状态

# --- 查看 Airbyte 版本 ---
abctl version
# 当前版本：v0.30.4，应用版本：2.0.1

# --- 手动触发数据同步 ---
# 在 Airbyte Web UI (http://localhost:8000) 中：
# Connections → 选择连接 → Sync Now

# --- 备用方案：不依赖 Airbyte 的直接同步 ---
cd /home/zp/airbyte/docker-platform
./run-pipeline.sh sync
# 通过 SQL COPY 直接从 source-postgres 同步到 dest-postgres

# --- 停止 ---
docker stop airbyte-abctl-control-plane
```

### 4.2 PostgreSQL — 数据库管理

```bash
# --- 连接源数据库 ---
docker exec -it source-postgres psql -U postgres -d power

# --- 连接目标仓库 ---
docker exec -it dest-postgres psql -U postgres -d warehouse
# 或从宿主机连接：
psql -h localhost -p 5433 -U postgres -d warehouse

# --- 查看表列表 ---
docker exec dest-postgres psql -U postgres -d warehouse -c "\dt public.*"

# --- 查看表行数 ---
docker exec dest-postgres psql -U postgres -d warehouse -c \
  "SELECT tablename, n_live_tup FROM pg_stat_user_tables ORDER BY n_live_tup DESC;"

# --- 查看数据库大小 ---
docker exec dest-postgres psql -U postgres -d warehouse -c \
  "SELECT pg_size_pretty(pg_database_size('warehouse'));"

# --- 查看 dbt 模型产生的表/视图 ---
docker exec dest-postgres psql -U postgres -d warehouse -c \
  "SELECT table_name, table_type FROM information_schema.tables
   WHERE table_schema='public' ORDER BY table_name;"

# --- 导出数据 ---
docker exec dest-postgres pg_dump -U postgres warehouse > backup_warehouse.sql

# --- 重启数据库 ---
docker restart source-postgres
docker restart dest-postgres
```

### 4.3 MinIO — 对象存储管理

```bash
# --- 启动 ---
docker start minio
# 控制台：http://localhost:9001 (minioadmin/minioadmin)

# --- 查看 bucket ---
docker exec minio mc ls local/
docker exec minio mc ls local/mlflow-artifacts/

# --- 检查存储用量 ---
docker exec minio mc du local/mlflow-artifacts/

# --- 重启 ---
docker restart minio
```

### 4.4 MLflow — 实验追踪管理

```bash
# --- 启动 ---
docker start mlflow
# Web UI：http://localhost:5002

# --- 查看实验列表 (命令行) ---
docker exec mlflow mlflow experiments search

# --- 查看某次运行的指标 ---
# 在 Web UI 中更方便：http://localhost:5002 → 选择实验 → 查看 Metrics

# --- 重启 ---
docker restart mlflow
```

### 4.5 Metabase — BI 看板管理

```bash
# --- 启动 ---
docker start metabase
# Web UI：http://localhost:3000

# --- 首次初始化后添加数据源 ---
# 设置 → Admin → Databases → Add Database
# 类型: PostgreSQL
# Host: dest-postgres | Port: 5432 | Database: warehouse
# Username: postgres | Password: postgres

# --- 查看日志（排查问题时） ---
docker logs metabase --tail 100

# --- 重启 ---
docker restart metabase
```

### 4.6 Ollama — 本地大模型管理

```bash
# --- 启动 ---
docker start ollama

# --- 查看已下载的模型 ---
docker exec ollama ollama list

# --- 下载新模型 ---
docker exec ollama ollama pull qwen2.5:7b
docker exec ollama ollama pull llama3:8b

# --- 测试对话 ---
docker exec ollama ollama run qwen2.5:1.5b "你好，请介绍一下自己"

# --- API 调用 ---
curl http://localhost:11434/api/generate \
  -d '{"model":"qwen2.5:1.5b","prompt":"你好"}'

# --- 删除模型（释放空间） ---
docker exec ollama ollama rm <model-name>

# --- 重启 ---
docker restart ollama
```

## 六、dbt 数据建模操作

### 6.1 基本命令

```bash
cd /home/zp/airbyte/power_warehouse

# 测试数据库连接
dbt debug

# 运行所有模型（20个 SQL 模型）
dbt run

# 运行指定层
dbt run --select staging.*          # 仅清洗层（7个模型）
dbt run --select intermediate.*     # 仅中间层（2个模型）
dbt run --select mart.*             # 仅集市层（3个模型）
dbt run --select features.*         # 仅特征层（3个模型）-- ML训练依赖此层
dbt run --select dashboard.*        # 仅仪表盘层（4个模型）

# 运行单个模型
dbt run --select feat_ml_features

# 运行模型及其所有上游依赖
dbt run --select +feat_ml_features

# 查看模型文档
dbt docs generate && dbt docs serve
```

### 6.2 模型层级说明

| 层级 | 模型数 | 说明 | 产出表 |
|------|--------|------|--------|
| **staging** | 7 | 原始数据清洗、类型转换、无效日期处理 | stg_hourly_load, stg_hourly_generation 等 |
| **intermediate** | 2 | 供需平衡计算、预测准确度 | int_hourly_supply_demand, int_forecast_accuracy |
| **mart** | 3 | 业务级日度汇总 | mart_daily_power/price_summary, mart_weather_power_correlation |
| **features** | 3 | 76列 ML 特征（滞后、滚动窗口等） | feat_hourly_base, feat_ml_features, feat_price_decomposition |
| **dashboard** | 4 | BI 看板数据视图 | dash_daily_overview, dash_hourly_comparison 等 |

### 6.3 修改模型后的操作

```bash
# 修改某个 .sql 模型文件后：
cd /home/zp/airbyte/power_warehouse

# 运行被修改的模型及其下游
dbt run --select <model_name>+

# 验证数据
dbt test

# 查看编译后的 SQL（调试用）
dbt compile --select <model_name>
cat target/compiled/power_warehouse/models/<layer>/<model_name>.sql
```

### 6.4 添加新数据源

1. 编辑 `models/staging/_sources.yml`，添加新表定义
2. 在 `models/staging/` 下创建对应的 `stg_<table>.sql`
3. 按需更新 intermediate / mart / features 层
4. 运行 `dbt run`

### 7.1 模型架构

P2 集成预测模型（电力现货价格预测）：

| 模型 | 权重 | 算法 | 说明 |
|------|------|------|------|

| LightGBM | 25% | LightGBM | 高效梯度提升 |
| XGBoost | 15% | XGBoost | 极端梯度提升 |
| Spread-GBR | 35% | GradientBoosting | 针对日前-实时价差 |
| BiLSTM | 5% | 双向LSTM | 序列特征捕捉 |

### 7.2 训练模型

```bash
cd /home/zp/airbyte/power_warehouse/ml

# - MinIO: http://localhost:9001 → mlflow-artifacts bucket 查看模型文件
```

```bash
cd /home/zp/airbyte/power_warehouse/ml
```


### 7.5 重新训练模型

当数据更新后，需要重新训练：

```bash
# 1. 确保新数据已同步
./run-pipeline.sh sync

# 2. 重新运行特征工程
```

## 九、故障排查与常见问题

### 9.1 服务无法访问

```bash
# 检查容器是否运行
docker ps -a --format "table {{.Names}}\t{{.Status}}"

# 查看容器日志
docker logs <container-name> --tail 50

# 检查端口占用
ss -tlnp | grep <port>

# 重启单个服务
docker restart <container-name>
```

### 9.2 Airbyte 返回 502

```bash
# Airbyte 启动需 3-5 分钟，耐心等待
docker exec airbyte-abctl-control-plane kubectl get pods -A
# 等所有 pod 状态变为 Running
```

### 9.3 数据库连接失败

```bash
# 检查容器健康状态
docker inspect --format='{{.State.Health.Status}}' source-postgres
docker inspect --format='{{.State.Health.Status}}' dest-postgres

# 手动测试连接
docker exec source-postgres pg_isready -U postgres -d power
docker exec dest-postgres pg_isready -U postgres -d warehouse
```

### 9.4 Docker 重启后容器没恢复

```bash
# docker-compose 管理的容器设置了 restart: unless-stopped
# 但如果是手动 docker run 创建的容器，需要手动启动：
docker start source-postgres dest-postgres minio mlflow metabase ollama

# 或统一用 compose：
cd /home/zp/airbyte/docker-platform && docker compose up -d
```

### 9.5 磁盘空间不足

```bash
# 查看磁盘使用
df -h

# 查看 Docker 空间占用
docker system df

# 清理未使用的 Docker 资源（谨慎！）
docker system prune        # 清理停止的容器、悬空镜像
docker volume prune        # 清理未使用的 volume（危险！会丢数据！）

# 清理 dbt 编译缓存
cd /home/zp/airbyte/power_warehouse && rm -rf target/ dbt_packages/
```

### 9.6 MLflow 无法上传 artifacts

```bash
# 确认 MinIO 正常
curl http://localhost:9000/minio/health/live

# 确认 bucket 存在且为 public
docker exec minio mc ls local/mlflow-artifacts/
docker exec minio mc anonymous get local/mlflow-artifacts
# 如果不是 public，重新设置：
docker exec minio mc anonymous set public local/mlflow-artifacts
```

## 十一、新机器部署指南

### 前置条件

```bash
# 1. Ubuntu 22.04+ / 16GB RAM / 50GB+ 磁盘
# 2. 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER

# 3. 安装 Anaconda (可选但推荐)
wget https://repo.anaconda.com/archive/Anaconda3-2024.10-1-Linux-x86_64.sh
bash Anaconda3-*.sh

# 4. 安装 abctl (Airbyte CLI)
curl -LsfS https://get.airbyte.com | bash -

# 5. 安装 dbt
pip install dbt-postgres

# 6. 中国大陆注意：配置 DNS 避免 Docker Hub 拉取失败
sudo mkdir -p /etc/systemd/resolved.conf.d
sudo tee /etc/systemd/resolved.conf.d/dns.conf << 'EOF'
[Resolve]
DNS=8.8.8.8 8.8.4.4
FallbackDNS=1.1.1.1
EOF
sudo systemctl restart systemd-resolved
sudo systemctl restart docker
```

### 部署步骤

```bash
# 1. 复制项目文件到新机器
scp -r /home/zp/airbyte/ user@new-machine:/home/user/airbyte/

# 2. 安装 Airbyte
abctl local install
# 等待完成，验证 http://localhost:8000

# 3. 启动数据平台
```

## 十二、踩坑速查表

| # | 问题 | 原因 | 解决方案 |
|---|------|------|---------|
| 1 | Airbyte 502 错误 | k8s pod 未就绪 | 等待 3-5 分钟 |
| 2 | SQLite dump 导入 PG 失败 | 语法差异 (PRAGMA/引号/自增) | sed 批量替换（init-source.sh 已处理） |
| 3 | `DEFAULT "全区域"` 建表失败 | PG 双引号=列名 | 改用单引号 |
| 4 | 无效日期 20250631 | 源数据质量 | dbt safe_date 宏自动处理 |
| 5 | Docker Hub 拉取失败 | DNS 污染 (中国大陆) | 修改 systemd-resolved DNS |
| 6 | TLS 证书错误 | auth.docker.io DNS 被劫持 | /etc/hosts 或换 DNS |
| 7 | kind load 找不到 | 只安装了 abctl | `docker save \| ctr images import` |
| 8 | Docker 重启后容器停止 | daemon restart | docker start 或 compose up -d |
| 9 | Metabase 连接失败 | Host 字段含空格 | 清除空格 |
| 10 | MLflow 端口冲突 | 5000/5001 已占用 | 已映射到 5002 |
| 11 | S3 artifact 上传拒绝 | MinIO bucket 权限 | mc anonymous set public |
| 12 | NumPy 版本不兼容 | pip/conda 混装 | 升级 pyarrow>=23 |
| 13 | psycopg2 不识别 np.float64 | numpy 类型非原生 Python | float() 包装 |
| 14 | API 缺列错误 | feat_ml_features 不含原始列 | JOIN feat_hourly_base |
