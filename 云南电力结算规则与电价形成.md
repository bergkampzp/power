# 云南电力市场结算规则与电价形成机理（建模纲领）

**版本**: V1.0 ｜ **日期**: 2026-06-05
**来源**: 《云南电力市场现货电能量交易实施细则》《云南电力市场结算实施细则》（云南能源局发布，征求意见稿）+ 发改价格〔2025〕136号
**用途**: 厘清"电价由什么决定、结算如何进行"，作为电价预测建模与数据采集的纲领。

> ⚠️ 关键修正（2026-06-05，用户拍板）：此前"水价决定电价水平"的说法**不准确**。正确机理见 §3——
> **风光优先消纳；出清价由边际机组（枯季通常是火电）的报价决定；水电用库容做时间优化，是边界约束/价格接受者，决定不了出清价。**

---

## 0. 一句话核心

**风光优先 → 残余负荷由水电/火电承担 → 出清价 = 边际机组报价（枯季=火电变动成本）→ 水电靠库容选择发电时间但不设定出清价。**

---

## 1. 市场体系与衔接

| 子市场 | 角色 | 与现货的衔接 |
|---|---|---|
| 中长期（年/月/周/多日） | 主体，覆盖绝大部分电量 | **差价合约结算**（CfD），开市前分解为分时电量（1h间隔） |
| 现货（日前 + 实时） | 结"偏差/价差" | SCUC/SCED 集中出清 |
| 调频等辅助服务 | 分步衔接 | SCUC 机组组合协同 |
| 跨省（南方区域） | 云南为送端 | 跨省与省内**联合出清**，含跨省优先发电计划/中长期 |

中长期合同（含电网代购、跨省优先发电计划、跨省中长期）**均通过差价合约与现货衔接**。

---

## 2. 现货出清与定价机制

- **方法**：以**社会福利最大化**为目标，用 **SCUC（安全约束机组组合）+ SCED（安全约束经济调度）** 集中优化，出清得到：机组开机组合、**分时出力计划、分时节点电价、跨省送受电计划**。
- **粒度**：运行日 D，每 **15 分钟一个出清时段（96 点）**；D-1 出日前，D 出实时，D+1 取实时结果。
- **定价**：**节点边际电价（LMP）**。起步阶段用户侧用**加权平均现货价（分区或全省统一结算价）**，成熟后用节点电价。
  - 👉 **这解释了我们数据里的 `__avg__`（全省统一/分区均价）与节点价并存**。
- **跨省送出价**：按南方区域现货市场出清结果确定。
- **边界约束**：含**水电厂水库运用约束、水电优化调度约束**、跨省送受电限额、非市场化机组安排、检修、电网安全约束。
  - 👉 **水库运用是"约束/优化变量"，不是定价机制**——印证 §3。
- **报价上限**：供应紧张/风险预警期，各类机组按 **变动成本 × 系数** 设申报价格上限（市场力缓解）。

---

## 3. 谁定价、谁是价格接受者（核心机理）

按《现货细则》§1.4 发电侧参与方式：

| 主体 | 申报方式 | 在出清中的角色 |
|---|---|---|
| **省内燃煤机组** | **报量报价** | **常为边际机组 → 报价（≈变动成本/煤价）决定出清价**（尤其枯季/尖峰） |
| **调节性能好的大型/梯级水电**（日以上调节水库、梯级流域大中型） | **报量报价** | 用**库容做时间优化**（高价时多发、低价时蓄水），实践中多为价格接受者；其报价反映机会成本但一般不设定边际价 |
| 径流式/调节差的水电、**风电、光伏** | **报量不报价** | **优先消纳、价格接受者**（近零成本，全额按出清价结算） |
| 用户（售电公司/批发用户） | **报量不报价**（起步阶段） | 需求曲线即中标曲线 |

**结论**：
1. **风光优先**（报量不报价、近零边际）→ 先被消纳，压低午间价、制造结构性低谷/零价。
2. **出清价由边际机组报价决定**；当残余负荷需要火电时，**火电变动成本（煤价驱动）= 出清价**。
3. **水电用库容做"时间选择"**（把水留到高价时段），影响**自己何时发**，但在边际定价机制下**不设定出清价**。

## 4. 结算公式（三段差价 / 全电量分段结算）

全电量 = 中长期净合约 + 日前偏差 + 实时偏差，**三段各按不同价格结算**（《结算细则》）：

**① 中长期（全电量按合约价）**
```
C_中长期 = Σ_t  Q_中长期净合约,t × P_中长期净合约综合价,t
```
按"分解后的净合约综合价"对中长期净合约电量做**全电量结算**；阻塞电费由双方约定，现阶段发电侧按机组物理节点、用户按统一结算点。

**② 日前偏差（日前电量 − 中长期合约电量，按日前价）**
```
C_日前 = Σ_t  (Q_日前,t − Q_中长期净合约,t) × P_日前统一结算点,t
```

**③ 实时偏差（实际上网/用电量 − 日前电量，按实时价）**
```
C_实时 = Σ_t  (Q_实际,t − Q_日前,t) × P_实时,t
```

**其他结算科目**：机组运行补偿/考核返还电费、用户侧偏差收益转移与中长期偏差收益回收、**市场不平衡资金**（跨区跨省 + 省内）、分摊未付款；跨省送电的省间/省内不平衡电费单独结算。

> 📌 **财务含义**：发电/用户的实际收入**不是裸现货价**，而是"中长期合约价 + 现货价差敞口"。**现货价波动只通过偏差电量影响收益**——这决定了预测的商业目标应是**价差敞口**，而非裸现货点价。

## 5. 与新能源机制电价（136号文）的衔接

《发改价格〔2025〕136号》：新能源原则上全部入市，建立**可持续发展价格结算机制**（机制电价 + 差价结算，多退少补）：市场价低于机制电价→补差，高于→扣差；存量（2025-06-01 前）按现行政策衔接，增量机制电价市场化竞价确定。

> 效果：新能源实际收益锁定在机制电价、**与现货价脱钩** → 新能源更倾向低价/零价抢发 → **进一步强化"风光优先 + 午间结构性低价"**，与 §3 一致，且使午间低价更"政策可预测"。

## 6. 对电价预测建模的含义（修正后）

| 维度 | 修正后的结论 |
|---|---|
| **价格水平由什么定** | **不是水价**，而是 **"残余负荷是否触及火电 + 火电变动成本（煤价）"**。残余负荷 = 负荷 − 风光 − 必开 − 径流水电 − 外送。 |
| **结构模型是否可行** | **重新可行**！定价者是火电、其变动成本相对可知 → 应建"**残余负荷 → 是否火电边际 → 火电变动成本**"的两段式供给模型（而非之前失败的纯净负荷曲线）。 |
| **水库/来水数据的定位** | 影响水电**何时发**（时间优化）与丰枯总量，**间接**影响残余负荷，但不直接定出清价 → 优先级低于煤价。 |
| **预测目标** | 节点/统一现货价用于运行；**商业目标是三段差价敞口**（中长期签约价、日前/实时偏差）。 |

### 数据需求（按本结算机理重排优先级）

| 优先级 | 数据 | 为什么（结算机理） |
|---|---|---|
| **P0** | **煤价 / 火电变动成本** | 火电是边际机组 → 直接定出清价水平 |
| **P0** | **残余负荷构成**：负荷、风光（优先消纳）、必开(非市场化)、外送曲线 | 决定是否触及火电边际 |
| **P1** | 火电可用容量、机组报价上限参数（变动成本×系数） | 供给栈与价格上限 |
| **P1** | 中长期分时合约价/电量、机制电价 | 三段差价敞口的预测目标 |
| **P2** | 水库蓄能/来水 | 水电时间优化与丰枯总量（间接） |

## 附：与本项目实测的相互印证

- 净负荷结构模型曾失败（rMAE 1.93）：**用错了定价者**——它假设净负荷直接定价，但实际定价者是**火电变动成本**。修正方向：把"火电是否边际 + 煤价"纳入，结构模型有望复活。
- GBDT 靠价格自回归取胜（rMAE 0.92）：价格历史隐含了煤价水平与残余负荷状态，故有效。
- 数据缺口由"水库/来水"**修正为"煤价/火电变动成本 + 残余负荷构成"** 为第一优先。
