# 电力市场数据数据库设计方案

## 一、数据源分析

### 1. 日前节点电价 (日前节点电价2-3月/)
- 文件格式: Excel (.xlsx)
- 数据结构: 每个文件包含当天96个时段(15分钟间隔)的节点电价
- 列: 地区 | 节点名称 | 时段1 ... 时段96
- 行数: 约1200+ 节点/天
- **表名**: `day_ahead_node_price`

### 2. 实时节点电价 (实时节点电价/)
- 文件格式: Excel (.xlsx)
- 数据结构: 同日前节点电价
- **表名**: `realtime_node_price`

### 3. 日前用电侧数据 (日前用电侧数据2-3月/)
- 文件格式: Excel (.xls)
- 数据内容: 日前交易结果-用电侧
- **表名**: `day_ahead_demand_result`

### 4. 实时用电侧数据
- 文件格式: Excel (.xls)
- **表名**: `realtime_demand_result`

### 5. 一键导出数据 (一键导出-2月/)
- 文件格式: ZIP压缩包 → 解压后Excel (.xls)
- 包含17类数据，约758个文件

#### 数据分类表:
| 数据类型 | 表名 | 说明 |
|---------|------|------|
| 云南参与优化出清水电机组群电量上限 | `day_ahead_hydro_unit_limit` | 机组出力比例、约束 |
| 高峰正备用信息 | `peak_reserve_info` | 高峰时段备用信息 |
| 稳定断面信息 | `stability_section_info` | 断面稳定信息 |

## 二、数据库架构设计

### 数据库名: power_market

```sql
-- =============================================
-- 核心表：日前节点电价
-- =============================================
CREATE TABLE day_ahead_node_price (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,           -- 交易日期
    region VARCHAR(50) NOT NULL,         -- 地区(云南)
    node_name VARCHAR(200) NOT NULL,     -- 节点名称
    period_00_15 DECIMAL(10,2),        -- 00:15时段电价
    period_00_30 DECIMAL(10,2),        -- 00:30时段电价
    period_00_45 DECIMAL(10,2),        -- 00:45时段电价
    period_01_00 DECIMAL(10,2),        -- 01:00时段电价
    -- ... 共96个时段 (每15分钟一个)
    period_23_45 DECIMAL(10,2),        -- 23:45时段电价
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- 索引: 日期+节点唯一约束
CREATE UNIQUE INDEX idx_da_node_price ON day_ahead_node_price(trade_date, node_name);
CREATE INDEX idx_da_price_date ON day_ahead_node_price(trade_date);
CREATE INDEX idx_da_price_node ON day_ahead_node_price(node_name);

-- =============================================
-- 核心表：实时节点电价
-- =============================================
CREATE TABLE realtime_node_price (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    region VARCHAR(50) NOT NULL,
    node_name VARCHAR(200) NOT NULL,
    period_00_15 DECIMAL(10,2),
    period_00_30 DECIMAL(10,2),
    -- ... 96个时段
    period_23_45 DECIMAL(10,2),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

CREATE UNIQUE INDEX idx_rt_node_price ON realtime_node_price(trade_date, node_name);
CREATE INDEX idx_rt_price_date ON realtime_node_price(trade_date);

-- =============================================
-- 日前用电侧交易结果
-- =============================================
CREATE TABLE day_ahead_demand_result (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    period VARCHAR(10),                  -- 时段
    demand DECIMAL(15,2),               -- 需求
    price DECIMAL(10,2),                -- 价格
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_da_demand_date ON day_ahead_demand_result(trade_date);

-- =============================================
-- 实时用电侧交易结果
-- =============================================
CREATE TABLE realtime_demand_result (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    period VARCHAR(10),
    demand DECIMAL(15,2),
    price DECIMAL(10,2),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_rt_demand_date ON realtime_demand_result(trade_date);

-- =============================================
-- 辅助表：水电机组群电量上限
-- =============================================
CREATE TABLE day_ahead_hydro_unit_limit (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    seq_no INTEGER,                     -- 序号
    region VARCHAR(50),                 -- 所属区域
    unit_group_name VARCHAR(200),       -- 机组群名称
    plant_name VARCHAR(200),            -- 电厂名称
    unit_name VARCHAR(200),             -- 机组名称
    ratio DECIMAL(10,4),               -- 机组出力占比
    effect_time TIMESTAMP,              -- 生效时间
    expire_time TIMESTAMP,              -- 失效时间
    power_constraint DECIMAL(15,2),     -- 电力约束
    energy_constraint DECIMAL(15,2),     -- 电量约束
    max_mode_constraint VARCHAR(10),     -- 最大运行方式约束
    min_mode_constraint VARCHAR(10),     -- 最小运行方式约束
    max_energy DECIMAL(15,2),           -- 最大电量
    min_energy DECIMAL(15,2),           -- 最小电量
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_hydro_limit_date ON day_ahead_hydro_unit_limit(trade_date);

-- =============================================
-- 高峰正备用信息
-- =============================================
CREATE TABLE peak_reserve_info (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    period VARCHAR(10),                  -- 时段
    reserve_type VARCHAR(50),            -- 备用类型
    reserve_value DECIMAL(15,2),        -- 备用值
    unit VARCHAR(20),                   -- 单位
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_peak_reserve_date ON peak_reserve_info(trade_date);

-- =============================================
-- 稳定断面信息
-- =============================================
CREATE TABLE stability_section_info (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    section_name VARCHAR(200),           -- 断面名称
    direction VARCHAR(50),               -- 方向
    stability_limit DECIMAL(15,2),       -- 稳定限额
    unit VARCHAR(20),                   -- 单位
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_stability_date ON stability_section_info(trade_date);

-- =============================================
-- 断面约束及阻塞情况
-- =============================================
CREATE TABLE section_constraint (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    market_type VARCHAR(20),             -- '日前' 或 '实时'
    section_name VARCHAR(200),
    constraint_type VARCHAR(50),        -- 约束类型
    limit_value DECIMAL(15,2),
    actual_value DECIMAL(15,2),
    congestion_degree VARCHAR(20),       -- 阻塞程度
    period VARCHAR(10),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_section_constraint_date ON section_constraint(trade_date);

-- =============================================
-- 重要通道输电情况
-- =============================================
CREATE TABLE transmission_channel (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    channel_name VARCHAR(200),
    power_flow DECIMAL(15,2),
    capacity DECIMAL(15,2),
    utilization_rate DECIMAL(10,4),
    period VARCHAR(10),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_transmission_date ON transmission_channel(trade_date);

-- =============================================
-- 电网负荷总体情况
-- =============================================
CREATE TABLE grid_load_overview (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    period VARCHAR(10),
    total_load DECIMAL(15,2),           -- 总负荷
    peak_load DECIMAL(15,2),            -- 峰值负荷
    valley_load DECIMAL(15,2),          -- 谷值负荷
    load_rate DECIMAL(10,4),            -- 负荷率
    unit VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_grid_load_date ON grid_load_overview(trade_date);

-- =============================================
-- 出力信息汇总表
-- =============================================
CREATE TABLE output_summary (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    output_type VARCHAR(50) NOT NULL,   -- '统调负荷', '发电总出力', '非市场化', '新能源', '水电'
    period VARCHAR(10),
    value DECIMAL(15,2),
    unit VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_output_date ON output_summary(trade_date);
CREATE INDEX idx_output_type ON output_summary(output_type);

-- =============================================
-- 检修计划
-- =============================================
CREATE TABLE maintenance_plan (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    unit_name VARCHAR(200),
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    description TEXT,
    status VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_maintenance_date ON maintenance_plan(trade_date);

-- =============================================
-- 机组状态
-- =============================================
CREATE TABLE unit_status (
    id SERIAL PRIMARY KEY,
    trade_date DATE NOT NULL,
    unit_name VARCHAR(200),
    unit_type VARCHAR(50),              -- 机组类型
    status VARCHAR(20),                 -- 状态
    capacity DECIMAL(15,2),
    available_capacity DECIMAL(15,2),
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_unit_status_date ON unit_status(trade_date);
```

## 三、分析框架建议

### 1. 数据分析框架选择

| 框架 | 适用场景 | 推荐度 |
|------|----------|--------|
| **PostgreSQL + TimescaleDB** | 时序数据存储/查询(推荐) | ⭐⭐⭐⭐⭐ |
| **Python + Pandas** | 数据清洗、处理 | ⭐⭐⭐⭐⭐ |
| **Apache Spark** | 大规模数据处理 | ⭐⭐⭐⭐ |
| **Dask** | 并行计算 | ⭐⭐⭐⭐ |
| **SQLAlchemy** | ORM映射 | ⭐⭐⭐⭐ |

**推荐技术栈**: PostgreSQL + TimescaleDB (时序数据库扩展) + Python

### 2. 电价预测方案

#### A. 机器学习方案

```
推荐技术栈:
├── 数据处理: Pandas, NumPy
├── 特征工程: Scikit-learn
├── 模型训练: 
│   ├── 传统ML: XGBoost, LightGBM, Random Forest
│   ├── 深度学习: LSTM, Transformer, Prophet
│   └── 集成学习: Stacking
├── 时序分析: Statsmodels, Prophet
└── 可视化: Matplotlib, Plotly, Streamlit
```

#### B. 预测模型建议

1. **基础模型** (入门):
   - 线性回归 + 特征工程
   - ARIMA/SARIMA
   
2. **进阶模型** (生产):
   - XGBoost/LightGBM + 时序特征
   - LSTM 神经网络
   
3. **高级方案** (研究):
   - Transformer 时序预测
   - 多模型集成 (Stacking)

#### C. 关键特征

| 特征类别 | 具体特征 |
|----------|----------|
| **历史电价** | 昨日同时段电价, 过去7天均值, 过去30天均值 |
| **负荷特征** | 统调负荷, 峰谷负荷, 负荷预测 |
| **新能源出力** | 光伏, 风电总出力 |
| **水电机组** | 水电总出力, 来水情况 |
| **备用信息** | 正备用, 负备用 |
| **网络约束** | 断面潮流, 阻塞情况 |
| **时间特征** | 星期, 是否节假日, 季节 |

## 四、电价预测分析方案

### 1. 分析维度

```
电价预测分析
│
├── 基础分析
│   ├── 电价分布统计 (均值、方差、分位数)
│   ├── 电价时序特征 (周期性、趋势)
│   └── 电价相关性分析
│
├── 影响因素分析
│   ├── 负荷与电价相关性
│   ├── 新能源出力与电价关系
│   ├── 水电出力影响
│   ├── 备用容量影响
│   └── 断面约束影响
│
├── 预测模型
│   ├── 短期预测 (日前 24-96小时)
│   ├── 超短期预测 (实时 1-6小时)
│   └── 节假日特殊模型
│
└── 模型评估
    ├── MAPE (平均绝对百分比误差)
    ├── RMSE (均方根误差)
    └── MAE (平均绝对误差)
```

### 2. 数据关联关系

```
┌─────────────────┐     ┌─────────────────┐
│  day_ahead_node │     │ realtime_node    │
│     _price      │     │     _price      │

         │                      │
         └──────────┬───────────┘
                    │
         ┌──────────▼───────────┐
         │   output_summary    │
         │ (负荷、出力数据)    │
         └──────────┬───────────┘
                    │
    ┌───────────────┼───────────────┐
    │               │               │
┌───▼───┐    ┌─────▼────┐    ┌─────▼────┐
│ reserve│    │ section  │    │  unit    │
│  _info │    │constraint│    │ _status  │
└───────┘    └──────────┘    └──────────┘
```

## 五、实施步骤

### 阶段1: 环境搭建 (1-2天)
1. 安装 PostgreSQL + TimescaleDB
2. 创建数据库和表结构
3. 配置数据库连接

### 阶段2: 数据入库 (1-2周)
1. 开发ETL脚本批量导入数据
2. 数据质量校验
3. 建立数据更新机制

### 阶段3: 数据清洗 (1周)
1. 缺失值处理
2. 异常值检测
3. 数据标准化

### 阶段4: 特征工程 (1-2周)
1. 构建时序特征
2. 构建外部特征(负荷, 出力, 备用)
3. 特征相关性分析

### 阶段5: 模型开发 (2-4周)
1. 基线模型
2. 进阶模型
3. 模型调优
4. 模型评估

### 阶段6: 部署上线 (1周)
1. 模型持久化
2. API服务开发
3. 定时预测任务
